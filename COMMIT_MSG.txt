Optimizer: fixed-size-array unrolling + redundant-cast cleanup, on a fast small corpus

## Dev infra: a self-contained small corpus (plato-src-small + Small/)

Renamed plato-spike-src → plato-src-small and rewrote it as a small, SELF-CONTAINED
corpus (Point3D/Bounds3D/Line3D built on Number, avoiding the System.Numerics-backed
Vector names) that reproduces the optimizer-relevant geometry shapes without the full
library. New Small/ tree builds two projects — Plato.Small.Generated.Unoptimized and
Plato.Small.Optimized — against a tiny hand-written Plato.Small.Runtime (MakeArray /
CombineHashCodes / eager IReadOnlyList combinators / Number-Integer-Boolean stubs),
NOT Plato.Intrinsics.V2. regen-small.ps1 regenerates + builds both. A seconds-fast
optimizer-spike loop, kept out of check-all.

## Optimizer improvements (all gated on the existing flags)

1. Component unroller accepts CHEAP-PROJECTION vector sources (`x.A`, not only bare
   params), reusing TirInliner.IsCheapProjection. Line3D.Eval (`x.A.Lerp(x.B, t)`) now
   emits `new Point3D(...)` instead of a Components()+Zip loop.

2. Fixed-size array unrolling. The inliner now inlines array-LITERAL bodies (Corners),
   and the component unroller unrolls Map/Zip/Reduce/All/Any/Reverse over a fixed-size
   TirArray. Bounds3D.Deform goes from a Corners()+`new Point3D[8]`+loop to
   `MakeArray<Point3D>(f.Invoke(c0)..f.Invoke(c7)).Bounds()` — no loop. TirArray now
   prints a typed `MakeArray<T>` so a per-element implicit conversion (Vector3→Point3D)
   survives the loop's removal.

3. Redundant-cast cleanup. A same-type `coerce<float→float>` whose inner is provably
   already that primitive (a native literal, an erased param that is not a wrapper-struct
   `this` receiver, a real erased component field, or a scalar-operator result) is
   dropped. Removes the `((float)x)` noise across every scalar-erased body; a
   wrapper-returning pseudo-field's cast is kept (it disambiguates a broadcast overload).

Not done: broad `var _varN = p;` copy-let removal — the Eval case it was raised for is
already fixed by (1); eliminating the rest means changing the shared lambda-capture hoist,
which the out-of-scope legacy TS/Rust writers still mirror byte-for-byte. Deferred.

Goldens refreshed (both recipes — the cast cleanup touches the unoptimized output too);
both projects build on net8.0. Emit snapshot re-baselined. New ComponentUnrollerTests
cover the Eval/fixed-array cases.

Gates: PlatoTests 116/116; conformance 205/205; both goldens + both Small projects build.
